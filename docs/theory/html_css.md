# 1. Теория - HTML и CSS

## **1.1. HTML: полный теоретический конспект на основе проекта**

### **1.1.1. Назначение HTML в веб-приложении**

**HTML (HyperText Markup Language)** — язык разметки, определяющий **структуру документа** и семантику содержимого веб-страницы.

В данном проекте HTML отвечает за:

* формирование структуры пользовательского интерфейса;
* описание форм ввода данных;
* логическую группировку элементов;
* создание таблиц данных;
* связывание с CSS и JavaScript.

HTML **не содержит бизнес-логики** и **не отвечает за внешний вид**, а задаёт каркас интерфейса.

---

### **1.1.2. Структура HTML-документа (HTML5)**

Каждый файл проекта построен по стандарту HTML5:

```html
<!doctype html>
<html lang="ru">
  <head>...</head>
  <body>...</body>
</html>
```

#### Теоретическое значение:

* `<!doctype html>` — включает стандартный режим браузера;
* `<html lang="ru">` — язык документа (важно для доступности и SEO);
* `<head>` — метаданные и подключения;
* `<body>` — визуальный и логический контент страницы.

---

### **1.1.3. Блок `<head>`: метаданные и подключения**

В проекте используются следующие элементы:

* `<meta charset="utf-8">` — поддержка кириллицы;
* `<meta name="viewport">` — адаптивность под мобильные устройства;
* `<title>` — заголовок вкладки браузера;
* `<link rel="stylesheet">` — подключение CSS;
* `<script src="...">` — подключение JavaScript.

#### Теория:

`<head>` **не отображается пользователю**, но определяет,
**как браузер интерпретирует документ**.

---

### **1.1.4. Семантика и иерархия заголовков**

Используются:

* `<h1>` — главный заголовок страницы;
* `<h2>` — заголовки разделов.

#### Теоретически:

* заголовки формируют логическую структуру;
* используются поисковыми системами;
* помогают навигации и доступности.

**Правило:** `<h1>` — один на страницу.

---

### **1.1.5. Контейнеры и группировка (`div`)**

Основные контейнеры проекта:

* `.container` — основной макет страницы;
* `.card` — логически завершённый блок;
* `.row`, `.field`, `.actions` — структурные элементы формы.

#### Теория:

`<div>` — универсальный контейнер для группировки,
семантика задаётся **через классы и контекст использования**.

---

### **1.1.6. Формы (`form`) и элементы ввода**

#### Используемые элементы:

* `<form>`
* `<input>`
* `<select>`
* `<textarea>`
* `<button>`
* `<label>`

#### Атрибуты HTML5:

* `required`
* `minlength`
* `pattern`
* `type`
* `placeholder`
* `name`
* `id`

#### Теория:

HTML-форма — **основа взаимодействия пользователя с системой**.
Она задаёт:

* контракт данных;
* первичную валидацию;
* структуру отправки данных.

---

### **1.1.7. Связывание `label` и элементов ввода**

Используется:

```html
<label for="id">...</label>
<input id="id">
```

#### Теоретическое значение:

* улучшение доступности;
* возможность фокуса по клику на текст;
* соответствие стандартам WCAG.

---

### **1.1.8. Таблицы (`table`) для отображения данных**

Используются элементы:

* `<table>`
* `<thead>`
* `<tbody>`
* `<tr>`
* `<th>`
* `<td>`

#### Теория:

Таблицы применяются **только для табличных данных**,
а не для верстки.

В проекте таблицы используются для:

* списка заявок;
* административного просмотра данных.

---

### **1.1.9. Навигация и гиперссылки**

Используется:

```html
<a href="/route">...</a>
```

#### Теория:

* `<a>` — навигация между ресурсами;
* связывает страницы в единое приложение;
* работает совместно с серверной маршрутизацией Flask.

---

### **1.1.10. Доступность (Accessibility) в HTML**

В проекте применяются:

* `lang="ru"`
* `alt` у изображений;
* `aria-label`
* `aria-live`
* `role="status"`

#### Теория:

HTML должен быть доступен:

* для скринридеров;
* для пользователей с ограничениями;
* для автоматических анализаторов.

---

### **1.1.11. Связь HTML с CSS и JavaScript**

HTML:

* подключает CSS через `<link>`;
* подключает JS через `<script>`;
* предоставляет DOM для управления из JavaScript.

HTML **никогда не содержит логику**, а лишь предоставляет точки управления.

---

## **2.1. CSS: полный теоретический конспект на основе проекта**

### **2.1.1. Назначение CSS**

**CSS (Cascading Style Sheets)** — язык описания внешнего вида HTML-документа.

В проекте CSS отвечает за:

* визуальное оформление;
* адаптивность;
* состояния элементов;
* микроанимации;
* единый стиль интерфейса.

---

### **2.1.2. Архитектура CSS-файла**

Файл `styles.css` реализует:

* базовые стили;
* дизайн-систему;
* адаптивную верстку;
* компонентный подход.

CSS централизован → изменения применяются ко всем страницам.

---

### **2.1.3. CSS-переменные и дизайн-система**

Используется `:root` с переменными:

```css
--bg, --text, --border, --focus, --danger, --ok
```

#### Теория:

* единая цветовая схема;
* масштабируемость;
* поддержка dark/light тем в будущем.

---

### **2.1.4. Сброс и базовые правила**

```css
* { box-sizing: border-box; }
```

#### Теория:

* предсказуемая модель размеров;
* предотвращение ошибок верстки;
* стандарт современной разработки.

---

### **2.1.5. Типографика и базовый стиль**

Определены:

* шрифты;
* цвет текста;
* отступы;
* размеры элементов.

#### Теория:

CSS управляет **читаемостью и визуальной иерархией**.

---

### **2.1.6. Mobile-first подход**

По умолчанию:

* стили под мобильные устройства;
* затем `@media (min-width: ...)`.

#### Теория:

Mobile-first:

* быстрее загружается;
* лучше масштабируется;
* соответствует современным стандартам.

---

### **2.1.7. CSS Grid и Flexbox**

Используются:

* `display: grid` — формы;
* `display: flex` — кнопки и действия.

#### Теория:

* Grid — двумерная компоновка;
* Flexbox — одномерное выравнивание.

---

### **2.1.8. Формы и состояния элементов**

Стили для:

* обычного состояния;
* `:focus`;
* ошибок (`.is-error`);
* успеха (`.is-ok`).

#### Теория:

Пользователь **должен визуально понимать**,
корректно ли он заполняет данные.

---

### **2.1.9. Таблицы и адаптивность**

Таблица:

* превращается в `block` на мобильных;
* получает горизонтальный скролл;
* возвращается в обычный вид на десктопе.

#### Теория:

Данные должны быть читаемы **на любом экране**.

---

### **2.1.10. Анимации и микроанимации**

Используются:

* `transition`;
* плавные изменения opacity и transform.

#### Теория:

Микроанимации:

* повышают UX;
* помогают понять изменения интерфейса;
* не отвлекают пользователя.

---

### **2.1.11. Toast-уведомления**

CSS управляет:

* позиционированием;
* анимацией появления;
* состояниями успех/ошибка.

#### Теория:

Toast — ненавязчивый способ обратной связи.

---

### **2.1.12. Связь CSS и JavaScript**

JavaScript:

* добавляет/удаляет CSS-классы;
* управляет состояниями;
* инициирует анимации.

CSS:

* отвечает за визуальную реакцию.

---

### **2.1.13. Ограничения текущего CSS**

Теоретически важно отметить:

* отсутствует препроцессор (SCSS);
* нет BEM/utility-фреймворка;
* нет CSS-модулей.

Для учебного проекта это допустимо и осознанно.
